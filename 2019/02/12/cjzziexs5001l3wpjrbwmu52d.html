JS网道学习笔记（十一）

发表于 2019-02-12 | 更新于 2019-02-28 | 分类于 JS网道学习笔记

1.严格模式

  • 启用方法：'use strict';
    • 老版本引擎会忽略，新版本就会进入
    • 可以用于整个脚本，也可以只用于单个函数
    • use strict放在函数体的第一行，则整个函数以严格模式运行
    • 可以在调用函数时，再传入”use strict”

示例如下

f2('"use strict";undeclared_variable = 1')

原本只用传入'undeclared_variable = 1'

  • 两个脚本合并在一起时，严格模式的脚本在前，则合并后的脚本都是严格模式；如果正常模式的脚本在前，则合并后的脚本都是正常模式

  • 常规模式对部分失败操作会不报错处理，而严格模式下这些地方就会报错

  • 函数不能有重名的参数
    • 正常模式下，如果函数有多个重名的参数，可以用arguments[i]读取。严格模式下，这属于语法错误
  • 禁止八进制的前缀0表示法
    • 正常模式下，整数的第一位如果是0，表示这是八进制数，比如0100等于十进制的64。严格模式禁止这种表示法，整数第一位为0，将报错
  • 严格模式下，使用eval或者arguments作为标识名，将会报错

下面的语句都会报错

'use strict';
var eval = 17;
var arguments = 17;
var obj = { set p(arguments) { } };
try { } catch (arguments) { }
function x(eval) { }
function arguments() { }
var y = function eval() { };
var f = new Function('arguments', "'use strict'; return 17;");
// SyntaxError: Unexpected eval or arguments in strict mode

  • 正常模式下，如果函数有多个重名的参数，可以用arguments[i]读取。严格模式下，这属于语法错误(出现重名参数就会报错)

  • 禁止八进制的前缀0表示法

    • 正常模式下，整数的第一位如果是0，表示这是八进制数，比如0100等于十进制的64。严格模式禁止这种表示法，整数第一位为0，将报错

  • 正常模式中，如果一个变量没有声明就赋值，默认是全局变量。严格模式禁止这种用法，全局变量必须显式声明(先声明再使用)

  • 正常模式下，函数内部的this可能会指向全局对象，严格模式禁止这种用法

    • 严格模式的函数体内部this是undefined
    • 这种限制对于构造函数尤其有用。使用构造函数时，有时忘了加new，这时this不再指向全局对象，而是报错
    • 严格模式下可以用call、apply和bind方法，将任意值绑定在this（undefined）上面，而正常模式下绑定时，this是先将绑定地值转为对象再绑定上去。而null和undefined这两个无法转成对象的值会被忽略

  • 严格模式下禁止使用 fn.callee、fn.caller

  • 严格模式下禁止使用arguments.callee、arguments.caller

  • 严格模式下无法删除变量，如果使用delete命令删除一个变量，会报错。只有对象的属性，且属性的描述对象的configurable属性设置为true，才能被delete命令删除

  • JavaScript 语言的一个特点，就是允许“动态绑定”，即某些属性和方法到底属于哪一个对象，不是在编译时确定的，而是在运行时（runtime）确定的

    • 严格模式对动态绑定做了一些限制。某些情况下，只允许静态绑定。也就是说，属性和方法到底归属哪个对象，必须在编译阶段就确定。这样做有利于编译效率的提高，也使得代码更容易阅读，更少出现意外

  • 以下几个方面为具体涉及

  • 严格模式下禁止使用with语句

  • 创设 eval 作用域

    • 正常模式下，eval语句的作用域，取决于它处于全局作用域，还是函数作用域。
    • 严格模式下，eval语句本身就是一个作用域，不再能够在其所运行的作用域创设新的变量了
      • 也就是说，eval所生成的变量只能用于eval内部

  • arguments 不再追踪参数的变化

  • 为了平稳过渡，严格模式引入了一些 ES6 语法

    • 以下为具体表现

  • 非函数代码块不得声明函数

    • ES6 会引入块级作用域。为了与新版本接轨，ES5 的严格模式只允许在全局作用域或函数作用域声明函数。也就是说，不允许在非函数的代码块（比如if和for）内声明函数

  • 严格模式下新增了一些保留字（implements、interface、let、package、private、protected、public、static、yield等）。使用这些词作为变量名将会报错

2.异步操作

  • JavaScript是单线程模型，引擎有多个县城，但是单个脚本只能在一个县城上运行（称为主线程），其他线程都是在后台配合

    • 常见的浏览器无响应（假死），往往就是因为某一段 JavaScript 代码长时间运行（比如死循环），导致整个页面卡在这个地方，其他任务无法执行
    • Javascript语言本身并不慢，慢的是读写外部数据
    • 设计者意识到这种情况，设置了”事件循环”机制，先挂起等待输入输出（IO操作）的任务，先运行排在后面的任务，等待IO操作有结果了，再回头继续任务

  • 程序里面的任务可以分为两类

    • 同步任务
      • 不挂起
    • 异步任务
      • 有挂起
      • 引擎认为某个异步任务可以执行了，该任务才会采用回调函数的形式进入主线程执行

  • 任务队列和事件循环

    • 除了主线程，引擎还提供一个任务队列，里面是需要当前程序处理的异步任务
    • 当一个同步任务执行完之后就去检查挂起来的异步任务，就调用可以执行异步任务的回调函数，这时这个异步任务就变成了一个同步任务
    • 如果一个异步任务没有回调函数，就不会进入任务队列，无法正常完成任务

  • 回调函数

    • 为了使f2在f1之后执行完成，可以让f2以回调函数的形式传给f1，然后再在f2中执行
    • 这样可以安排两个事件的执行顺序

  • 事件监听

    • 还可以采用事件驱动模式，比如利用jQuery的on方法等，让一个

  • 发布/订阅

    • 存在一个“消息”中心，任务执行完成，向信号中心发布信号，其他任务向信号中心订阅该信号
    • 具体操作结合相应模块
    • 该操作优于“事件监听”，因为可以查看“消息中心”从而更好监控程序运行

  • 异步操作的流程控制

第一个示例

function final(value) {
  console.log('完成: ', value);
}

async(1, function (value) {
  async(2, function (value) {
    async(3, function (value) {
      async(4, function (value) {
        async(5, function (value) {
          async(6, final);
        });
      });
    });
});
});
// 参数为 1 , 1秒后返回结果
// 参数为 2 , 1秒后返回结果
// 参数为 3 , 1秒后返回结果
// 参数为 4 , 1秒后返回结果
// 参数为 5 , 1秒后返回结果
// 参数为 6 , 1秒后返回结果
// 完成:  12

上面代码有六个回调函数的嵌套，写起来麻烦，容易出错，且难以维护

第二个示例，串行执行

var items = [ 1, 2, 3, 4, 5, 6 ];
var results = [];

function async(arg, callback) {
  console.log('参数为 ' + arg +' , 1秒后返回结果');
  setTimeout(function () { callback(arg * 2); }, 1000);
}

function final(value) {
  console.log('完成: ', value);
}

function series(item) {
  if(item) {
    async( item, function(result) {
      results.push(result);
      return series(items.shift());
    });
} else {
    return final(results[results.length - 1]);
}
}

series(items.shift());

series就是一个串行函数，它会依次执行异步任务，执行完后才调用final函数（总体也需要6秒）

第三个示例，并行执行

var items = [ 1, 2, 3, 4, 5, 6 ];
var results = [];

function async(arg, callback) {
  console.log('参数为 ' + arg +' , 1秒后返回结果');
setTimeout(function () { callback(arg * 2); }, 1000);
}

function final(value) {
  console.log('完成: ', value);
}

items.forEach(function(item) {
  async(item, function(result){
    results.push(result);
    if(results.length === items.length) {
      final(results[results.length - 1]);
    }
})
});

第四个示例，并行串行结合

var items = [ 1, 2, 3, 4, 5, 6 ];
var results = [];
var running = 0;
var limit = 2;

function async(arg, callback) {
  console.log('参数为 ' + arg +' , 1秒后返回结果');
  setTimeout(function () { callback(arg * 2); }, 1000);
}

function final(value) {
  console.log('完成: ', value);
}

function launcher() {
  while(running < limit && items.length > 0) {
    var item = items.shift();
    async(item, function(result) {
      results.push(result);
      running--;
      if(items.length > 0) {
        launcher();
     } else if(running == 0) {
      final(results);
      }
  });
  running++;
  }
}

launcher();

根据个人的理解，JavaScript里面的并行应该是对资源的一种合理分配，遇到并行操作时（比如定时器，比如Ajax）会先挂起没有占用资源的任务，接着往下进行

2.定时器

  • setTimeout函数用来指定某个函数或某段代码，在多少毫秒之后执行。它返回一个整数，表示定时器的编号，以后可以用来取消这个定时器。

示例如下

var timerId = setTimeout(func|'code', delay);

    • 除了前两个参数，setTimeout还允许更多的参数。它们（传入的回调函数的参数）将依次传入推迟执行的函数（回调函数）
    • 还有一个需要注意的地方，如果回调函数是对象的方法，那么setTimeout使得方法内部的this关键字指向全局环境，而不是定义时所在的那个对象

示例如下

var x = 1;

var obj = {
  x: 2,
  y: function () {
   console.log(this.x);
  }
};

setTimeout(obj.y, 1000) // 1

此时this的指向已经变化了

为了防止出现这个问题，一种解决方法是将obj.y放入一个函数

这样一来obj.y在obj的作用域执行，而不是在全局作用域内执行

另外一种是使用bind方法，将obj.y这个方法绑定在obj上面
