JS网道学习笔记（十）

发表于 2019-02-12 | 更新于 2019-02-28 | 分类于 JS网道学习笔记

1.Object对象的相关方法

  • Object.getPrototypeOf()方法返回参数对象的原型
    • 空对象的原型是 Object.prototype
    • Object.prototype 的原型是 null
    • 函数的原型是 Function.prototype
  • Object.setPrototypeOf()方法方法为参数对象设置原型，返回该参数对象。它接受两个参数，第一个是现有对象，第二个是原型对象

示例-使用Object.setPrototypeOf方法模拟new命令

var F = function () {
  this.foo = 'bar';
};

var f = new F();
// 等同于
var f = Object.setPrototypeOf({}, F.prototype);
F.call(f);
  • Object.create()方法接受一个对象作为参数，然后以它为原型，返回一个实例对象。该实例完全继承原型对象的属性
    • 如果想要生成一个不继承任何属性（比如没有toString和valueOf方法）的对象，可以将Object.create的参数设为null
    • Object.create方法生成的新对象，动态继承了原型。在原型上添加或修改任何方法，会立刻反映在新对象之上
    • Object.create方法还可以接受第二个参数。该参数是一个属性描述对象，它所描述的对象属性，会添加到实例对象，作为该对象自身的属性
  • Object.prototype.__proto__
    • 实例对象的proto属性（前后各两个下划线），返回该对象的原型。该属性可读写

示例如下

var obj = {};
var p = {};

obj.__proto__ = p;
Object.getPrototypeOf(obj) === p // true

__proto__本质是一个内部属性，不应该对使用者暴露，应该尽量少用这个属性

  • 获取实例对象的原型对象，有三种方法
    • obj.__proto__
    • obj.constructor.prototype
    • Object.getPrototypeOf(obj)

上面三种方法之中，前两种都不是很可靠。__proto__属性只有浏览器才需要部署，其他环境可以不部署。而obj.constructor.prototype在手动改变原型对象时，可能会失效（constructor要手动更改）。

  • Object.getOwnPropertyNames()方法返回一个数组，成员是参数对象本身的所有属性的键名，不包含继承的属性键名

    • 要返回不可以遍历的键名

  • Object.prototype.hasOwnProperty()方法返回一个布尔值，用于判断某个属性定义在对象自身，还是定义在原型链上

示例如下

Date.hasOwnProperty('length') // true
Date.hasOwnProperty('toString') // false

  • in运算符返回一个布尔值，表示一个对象是否具有某个属性

    • 不区分该属性是对象自身的属性，还是继承的属性

  • 对象的拷贝

    • 拷贝要做的两件事情
      • 相同原型
      • 同样的实例属性

最新示例如下

function copyObject(orig) {
  return Object.create(
    Object.getPrototypeOf(orig),
    Object.getOwnPropertyDescriptors(orig)
  );
}

Object.getOwnPropertyDescriptors方法用于返回一个对象的属性描述对象

这样做优于用assign，因为assign会无法正确拷贝set属性和get属性
