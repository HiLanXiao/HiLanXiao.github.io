JS网道学习笔记（九）

发表于 2019-02-08 | 更新于 2019-02-28 | 分类于 JS网道学习笔记

1.构造函数的缺点

  • 通过构造函数为实例对象定义属性，虽然很方便，但是有一个缺点。同一个构造函数的多个实例之间，无法共享属性，从而造成对系统资源的浪费
  • JavaScript 继承机制的设计思想就是将属性和方法定义在原型上，使所有实例对象能共享
    • JavaScript 规定，每个函数都有一个prototype属性，指向一个对象
      • 对于普通函数来说，该属性基本无用。但是，对于构造函数来说，生成实例的时候，该属性会自动成为实例对象的原型
      • 是共享的对象不是实例对象自身的属性，只要修改原型对象，变动就立刻会体现在所有实例对象上
      • 如果实例对象自身就有某个属性或方法，它就不会再去原型对象寻找这个属性或方法

2.原型链

  • “原型链”（prototype chain）：对象到原型，再到原型的原型……
    • 一层层地上溯，所有对象的原型最终都可以上溯到Object.prototype，即Object构造函数的prototype属性
    • Object.prototype的原型是null。null没有任何属性和方法，也没有自己的原型。因此，原型链的尽头就是null
    • 所寻找的属性在越上层的原型对象，对性能的影响越大
    • 如果让构造函数的prototype属性指向一个数组，就意味着实例对象可以调用数组方法

3.constructor 属性

  • prototype对象有一个constructor属性，默认指向prototype对象所在的构造函数

示例如下

function P() {}
P.prototype.constructor === P // true

constructor属性定义在prototype对象上面，意味着可以被所有实例对象继承

  • constructor属性的作用是，可以得知某个实例对象，到底是哪一个构造函数产生的
  • 另一方面，有了constructor属性，就可以从一个实例对象新建另一个实例

示例如下

function Constr() {}
var x = new Constr();

var y = new x.constructor();
y instanceof Constr // true

constructor属性表示原型对象与构造函数之间的关联关系，如果修改了原型对象，一般会同时修改constructor属性，防止引用的时候出错

function Person(name) {
  this.name = name;
}

Person.prototype.constructor === Person // true

Person.prototype = {
  method: function () {}
};

Person.prototype.constructor === Person // false
Person.prototype.constructor === Object // true

构造函数的原型对象修改后，constructor属性不再指向Person，因为新原型是一个普通对象，所以指向Object构造函数,所以修改原型对象时，一般要同时修改constructor属性的指向

  • 如果不能确定constructor属性是什么函数，还有一个办法：通过name属性，从实例得到构造函数的名称

4.instanceof运算符

  • instanceof检查整个原型链，检查右边原型链中是否存在左边实例对象的构造函数

下面两种写法是等价的

v instanceof Vehicle
// 等同于
Vehicle.prototype.isPrototypeOf(v)
  • instanceof运算符只能用于对象，不适用原始类型的值

5.构造函数的继承

  • 首先在子类的构造函数中，调用父类的构造函数

示例如下

function Sub(value) {
  Super.call(this);
  this.prop = value;
}

其中Super是父类的构造函数，this.prop应该是子类新增的属性

  • 再让子类的原型指向父类的原型

示例如下

Sub.prototype = Object.create(Super.prototype);
Sub.prototype.constructor = Sub;
Sub.prototype.method = '...';

上面代码中，Sub.prototype是子类的原型，要将它赋值为Object.create(Super.prototype)，而不是直接等于Super.prototype。否则后面两行对Sub.prototype的操作，会连父类的原型Super.prototype一起修改掉

另外一种写法是Sub.prototype等于一个父类实例。

Sub.prototype = new Super();

这种会包括符类实例的方法，所以可能并不是我们需要的，就不推荐这种写法

  • 有时只需要单个方法的继承，这是可以采用下面的写法

    ClassB.prototype.print = function() {

    ClassA.prototype.print.call(this);
// some code

    }

6.多重继承

  • JavaScript不允许一个对象同时继承多个对象，但是可以通过变通方法实现这个功能

示例如下

function M1() {
  this.hello = 'hello';
}

function M2() {
  this.world = 'world';
}

function S() {
  M1.call(this);
  M2.call(this);
}

// 继承 M1
S.prototype = Object.create(M1.prototype);
// 继承链上加入 M2
Object.assign(S.prototype, M2.prototype);

// 指定构造函数
S.prototype.constructor = S;

var s = new S();
s.hello // 'hello'
s.world // 'world'
s instanceof M1() // true
s instanceof M2() // false

过程是，在子类构造函数中调用M1和M2的构造方法，然后继承M1，再将M2原型合上去，注意后面原型会将前面原型同名部分覆盖

7.利用构造函数封装私有变量

  • 利用构造函数，封装私有变量

示例如下

function StringBuilder() {
  var buffer = [];

  this.add = function (str) {
    buffer.push(str);
  };

  this.toString = function () {
    return buffer.join('');
  };

}

注意这里的buffer前面没有使用this，是封装在构造函数中的私有变量
这意味着，构造函数既用来塑造了实例对象，又用来保存实例对象的数据，违背了构造函数与实例对象在数据上相分离的原则（即实例对象的数据，不应该保存在实例对象以外）。同时，非常耗费内存

8.利用立即执行函数的写法封装私有变量

示例如下

var module1 = (function () {
　var _count = 0;
　var m1 = function () {
　  //...
　};
　var m2 = function () {
　　//...
　};
　return {
　　m1 : m1,
　　m2 : m2
　};
})();

使用这种写法，外部代码无法读取内部的_count变量

立即执行函数指，声明了一个函数以后就立刻传入参数

9.模块的放大模式

var module1 = (function (mod){
　mod.m3 = function () {
　　//…
　};
　return mod;
})(module1);

在浏览器环境中，模块的各个部分通常都是从网上获取的，有时无法知道哪个部分会先加载。如果采用上面的写法，第一个执行的部分有可能加载一个不存在空对象，这时就要采用”宽放大模式”（Loose augmentation）

var module1 = (function (mod) {
　//...
　return mod;
})(window.module1 || {});

与”放大模式”相比，“宽放大模式”就是“立即执行函数”的参数可以是空对象

10.输入全局变量

  • 独立性是模块的重要特点，模块内部最好不与程序的其他部分直接交互
  • 为了在模块内部调用全局变量，必须显示地将其他变量输入模块

示例如下

var module1 = (function ($, YAHOO) {
　//...
})(jQuery, YAHOO);
