前端萌新成长计划&考核

新学期伊始，又到了一年一度的招新时间 我靠我的假期怎么就余额不足了

为了帮助各位同学尽快了解前端和能上手编写一些小demo，找到编程的乐趣 实际上是为了完成招新考核任务， 学长在这里给大家布置了一些小任务，如果大家对加入我们工作室感兴趣，可以尝试完成这些任务，并按照要求把相关成果发至指定邮箱

Step 1 养成良好习惯

在这个阶段没有任何实质性的考核，有的只是学长在学习习惯上的一些小建议（当然完成其中的小任务并按要求发到相应邮箱会加分哦）

  • 学会使用搜索引擎 && 询问学长学姐 && 一些网站
    • 从谷歌搜获更多 同样都是一台电脑，一个搜索引擎，为什么别人能快速定位问题，并找到解决方案？这个链接会给你解答部分疑惑
    • 知乎 知乎治校似已成我校传统，在这个网站你可以刷刷一些有趣的贴子，但同样，你也可以获得一些过来人的宝贵经验，不信，搜一搜 “前端 路线”？
    • PanDownload 这是一个神奇的下载工具，可以让你搜到百度云上一些非常有用的东西 用魔法打败魔法吧，少年 ，当然，在方便的同时，请你也同样尊重创造者的知识产权
    • 提问的智慧 遇到问题请先自行搜索引擎！不过当多次搜索无果，或者对搜索到的结果感到困惑时，请大方地询问 学长 学姐 吧
    • GitHub 全球最大男性同性交友平台，其精妙之处请自行知乎或谷歌
    • 思否 不介绍了，自己点开看吧。。。
    • 掘金 同上
    • stack overflow 中文搜索搜不到自己遇到的问题？ 用英文来这里搜搜！
  • 一个神奇的梯子
    • 什么是梯子？
      • 众所周知，由于GFW的存在，广大中国网民在访问许多国外网站时会受到限制，例如你点不开GFW这个链接，例如你用不上比某度搜索引擎好用的多的，Google搜索，例如你也上不了国外的一些官方网站等等。为了翻过这堵高墙，你可以选择购买vpn服务，当然也可以自己动手，租用一个国外的服务器，搭一个属于自己的，翻墙梯子
    • 为什么要翻墙？
      • 为了更优质的搜索引擎–Google
      • 为了外网上的一些优质资源
      • 看到更广阔的世界（Quora、YouTube等）
    • 任务（非强制）
      • 参考前端资料中的科学上网之用vultr搭建ss（卓然子出品），搭建一个属于自己的梯子
      • 写一篇10个字以上的总结，描述你遇到的问题和解决思路，讲一讲你在墙外看到了些啥有趣的东西，讲一讲学长的建议是否对你有帮助
      • 总结中需要有远程终端上的操作截图
      • 邮箱号为**1256925921@qq.com，邮件请命名为 姓名 昵称 + 学号**

Step2 加入全球最大同性交友网站 —— Github

从这一步开始，就是每位同学必须完成的任务了哦，想加入工作室的同学，请一定按要求发送相关成果至指定邮箱

  • 尝试食用 主流 分布式版本控制系统——Git

    • 什么是Git？

      • git是什么？如何使用git进行团队协作？

      • 如何优雅地使用 Git

      • Oh Shit，git！

      • 两个最常见的Git使用场景

        • 一个人开发：

          • 大佬这个代码🐮🍺，clone 一手到本地研究下

            （更便捷地学习开源代码）

          • 代码写完了，美滋滋，给自己倒一杯卡布奇洛，打开bash，开始把代码push到自己的git仓库 （相当于文件备份时把文件备份到了网盘上）

          • 代码写崩了，一堆bug不会改，反手一个超级时光机，reset一波回到 0 warning 0 error 的时候

            （版本控制，随时回到你想回到的时间点）

        • 多人合作开发：

          • 马飞飞开发模块A，卢伟伟开发模块B，各自先在自己的 branch上开发，马飞飞写好了自己的功能模块，测试确保没有问题了，再 merge到主分支；卢伟伟把系统写崩了，但是一点都不会影响到主分支的内容

    • 我该如何学习Git

      • Git教程 - 廖雪峰的官方网站
      • Learn Git Branching

  • 加入 Github大家庭

    • 或许有零基础的萌新会问， Github 和 Git 有什么关系，实际上Github只是采用了Git的一种仓库，诸如此类的仓库还有 Gitlab、码云等，但是Github由于其上有着众多的开源资源，和一代一代程序猿的心血传递，它更像是一个程序猿社区，而不是一个简单的代码仓库，你可以在上面提问和浏览别人的心得笔记，当然，你也可以做那个解答别人问题和发布心得，甚至是参与大型开源项目的巨佬。但这一切的起点，你得创建一个账号吧hhh

  • 任务（注：这一阶段的任务，请全部反映在 “博客” 里面）**

    • 任务一：创建一个Github账号，并配置好本机的SSH
      • 在这个阶段你需要搞清楚
        • 什么是协议
        • SSH是怎样一种协议
        • Git采用了哪些协议
        • 这些内容需要反映在你之后的博客里面
    • 任务二：创建一个仓库，并上传你的代码（任意代码）
      • 在这个阶段你需要做到
        • 代码至少要能跑对吧。。
        • 截图和 Github仓库证明你是独立完成的操作*
      • 需要练习以下操作：
        • 本地仓库的初始化与绑定远端仓库
        • 将任意代码push到master分支上
        • 创建一个新的分支，并将代码push到新的分支上
        • 将这个新的分支与master分支合并
        • 其他操作如代码回溯请自行练习，如有截图展示则加分
    • 任务三：创建一个新的仓库，并把你的第一个博客搭建在这个仓库上
      • 为什么采用Github Page + hexo 搭建第一个博客
        • 网上教程丰富
        • 无需手动编写代码
      • 推荐你的仓库命名结尾为.github.io，之后会少一些麻烦
      • 随便搜到的一个教程
      • 在这个阶段你需要做到
        • 喂，点开你发送的链接不能给个404对吧
        • 撰写你的第一篇博客
          • 博客内容
            • 请将Step1中的邮件内容转移到博客中
            • 回答任务一中的三个问题
            • 完成任务二中要求的操作，并给出相应的截图和心得
            • 总结你搭建博客中遇到的问题和如何解决的，没有遇到问题可给出大致的过程截图并附上简单的文字介绍
